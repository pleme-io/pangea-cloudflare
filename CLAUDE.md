Cloudflare provider resources for the Pangea infrastructure DSL. All resource
code is **auto-generated** by `pangea-forge` via `iac-forge-cli`. Manual edits
to resource files will be overwritten on the next generation run.

## Structure

```
lib/
  pangea-cloudflare.rb                  # Entry point (generated -- requires all resources)
  pangea-cloudflare/version.rb          # Version constant (manual)
  pangea/
    resources/
      types/cloudflare/
        core.rb                         # Core Cloudflare Dry::Types (generated)
        workers.rb                      # Workers-specific types (generated)
        security.rb                     # Security-specific types (generated)
        load_balancing.rb               # Load balancing types (generated)
      cloudflare.rb                     # Aggregator module (generated)
      cloudflare_*/                     # 200 resource directories (generated)
        types.rb                        # Dry::Struct attributes class
        resource.rb                     # ResourceBuilder DSL
spec/
  spec_helper.rb                        # Test setup (manual)
  support/                              # Shared test helpers
  resources/                            # RSpec stubs (generated)
```

## Generated vs Manual

| File | Source |
|------|--------|

| `lib/pangea-cloudflare.rb` | Generated (require file) |

| `lib/pangea/resources/types/cloudflare/` | Generated (type definitions) |

## Regeneration

Resources are generated by `pangea-forge` (the Pangea backend of `iac-forge`).
Run regeneration via `iac-forge-cli`:

```sh
terraform-schema-importer generate \
  --schema schema.json \
  --provider "registry.terraform.io/cloudflare/cloudflare" \
  --provider-name cloudflare \
  --output-dir .
```
